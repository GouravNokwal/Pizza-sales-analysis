
#  Pizza Sales Data Analysis

## Project Overview

This project focuses on analyzing pizza sales data to understand business performance and customer ordering patterns. The analysis is performed using Python for Exploratory Data Analysis (EDA), SQL for querying business metrics, and Power BI for interactive dashboard visualization.

The goal of this project is to extract meaningful insights such as total revenue, popular pizza categories, sales trends, and customer preferences to support data-driven decision making.

## Problem Statement

A pizza restaurant chain wants to analyze its historical sales data to understand customer purchasing behavior and overall sales performance. The management requires insights into key metrics such as total revenue, order volume, top-selling pizzas, and sales distribution across different pizza categories and sizes. The analysis aims to identify trends and provide actionable insights that help improve business strategies and profitability.

## Dataset Description

The dataset contains detailed information about pizza orders including:

* **pizza_id** – Unique identifier for each pizza item
* **order_id** – Unique identifier for each order
* **pizza_name_id** – Identifier for the pizza type
* **quantity** – Number of pizzas ordered
* **order_date** – Date of the order
* **order_time** – Time of the order
* **unit_price** – Price per pizza
* **total_price** – Total price of the order item
* **pizza_size** – Size of the pizza (S, M, L, XL, XXL)
* **pizza_category** – Category of pizza (Classic, Supreme, Veggie, Chicken)
* **pizza_ingredients** – Ingredients used in the pizza
* **pizza_name** – Name of the pizza

## Tools and Technologies Used

* **Python**
* **Pandas**
* **Matplotlib**
* **Seaborn**
* **SQL (PostgreSQL)**
* **Power BI**

## Exploratory Data Analysis (EDA)

EDA is performed using Python to understand the dataset and identify patterns.

Key steps include:

* Data loading and inspection
* Checking missing values and duplicates
* Data type correction
* Summary statistics analysis
* Outlier detection using boxplots
* Time-based analysis of orders
* Category and size distribution analysis

## SQL Analysis

SQL queries are used to extract business insights from the dataset.

Some key analyses include:

* Total revenue generated
* Total number of orders
* Top-selling pizzas
* Revenue by pizza category
* Revenue by pizza size
* Daily and hourly order trends

## Power BI Dashboard

An interactive dashboard is created in Power BI to visualize key metrics and business insights.

The dashboard includes:

* Total Revenue KPI
* Total Orders KPI
* Total Pizzas Sold KPI
* Sales by Pizza Category
* Sales by Pizza Size
* Top Selling Pizzas
* Order Trends Over Time

## Key Insights

* Certain pizza categories generate higher revenue compared to others.
* Medium and Large pizza sizes are the most frequently ordered.
* Sales show clear peak hours during lunch and dinner times.
* A small number of pizza types contribute significantly to overall sales.

## Conclusion

This project demonstrates an end-to-end data analysis workflow, starting from raw data exploration to business insight generation and interactive dashboard visualization. The analysis helps identify customer preferences, sales trends, and high-performing products, enabling better strategic decisions for the business.
